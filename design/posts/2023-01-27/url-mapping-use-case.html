The problem

Our sign-in and registration flow is complex.

Diagram showing the path users take to sign-in or register

Due to the nature of third-party sign-ins when a user clicks 'sign-in' or 'register' they're taken to the same web page to choose their third-party authentication provider. it's a challenge to deliver content that makes sense

How it affects users

Users lose trust and confidence.

It's confusing to arrive on a page and read something like 'sign-in or register' when you just indicated your intent by pressing either button.

A solution

Tweak the content on the page to suit people's intent.

GCWeb's URLMapping

[Use when] the page needs to be modified slightly to improve the content usability.

  • URLMapping Documentation (GCWeb)
  • URLMapping example (GCWeb)

How it works

Register and Sign-in buttons link to the same page but add a query string to pass what button was clicked to the sign-in and registration services page.

URLMapping allows us to read and act on the URL in the HTML mark-up with no custom scripting and while maintaing content in one place.
